Self-hosting & workspace directory file

This app is a single HTML file — drop it in your web root (e.g. public_html/rfp/index.html) and it runs as-is. To ship a shared knowledge base with it, export the workspace as countersign-data.json and place it in the same directory: the app auto-loads it on first open, so your whole team starts with the same 500-source bucket. On your own hosting, work also persists in each visitor's browser storage. To share one API key across the team instead of everyone pasting their own, deploy the bundled proxy.php (shared hosting) or proxy.js (Node) — see the README — and pick Team server proxy in AI settings.

Answer engine not connected

Run the bundled engine.js on a server (see ENGINE.md) to keep the knowledge base and the API key server-side. Pick Countersign Engine in ⚙ AI settings and generation switches to the server's retrieval — with whole-RFP batches streamed back live. Push this bucket up so the engine indexes the same sources.
